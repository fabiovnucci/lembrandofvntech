01. O Modelo de Caixa (Box Model) Tag: <section>

Margin vs Border vs Padding

O conceito mais importante para não quebrar o layout.

  • Margin: Espaço externo (afasta vizinhos).
  • Border: A linha de contorno.
  • Padding: Espaço interno (respiro do conteúdo).
margin: 20px; border: 4px solid #333; padding: 30px;

Vermelho = Margin | Azul/Verde = Padding

04. Fluxo de Trabalho (Git & GitHub) Tag: <section>

Do Zero ao Repositório

O roteiro exato para iniciar um projeto e manter o backup seguro no GitHub.

1. Preparação (Local):

  • git init: Cria o motor do Git na pasta.
  • git status: Lista o que foi criado ou alterado.
  • git add .: Prepara todos os arquivos para o "carimbo".

2. O Envio (Nuvem):

  • git commit -m "msg": Cria a versão com uma descrição.
  • git push origin main: Sobe o código para o GitHub.
git init git add . git commit -m "primeiro commit" git push origin main
# Iniciando o projeto FVN TECH
$ git init
Initialized empty Git repository...

$ git add .
$ git commit -m "feat: setup inicial"
[main (root-commit) 7a2b] feat: setup...

# O comando de emergência que usamos hoje:
$ git push origin HEAD:main --force
